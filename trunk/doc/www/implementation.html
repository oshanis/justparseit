Implementation of the Policy Parser


As briefly described in the Intro section, the Policy Parser consists of three major components: the NLTK Parser, the Policy Interpreter, and the RDF Generator. The system also includes the online RDF policy store as well as the AIR reasoner, but these can be considered as peripheral to the above three components. In this section, we describe each of these components in detail:


NTLK Parser: Given a user's input policy, the first step is to parse this sentence into a structured format that is manipulable by a computer. Automatically parsing any natural language sentence is an extremely challenging task, and is out of scope of this project. Thus, we focus on a set of constrained natural language sentences; more specfiically, our system is able to process a sentence of the following form:


subject [mod] action object ['to' secondary_object] [purpose] [condition]*


where:

  • subject: The main actor or entity who carries out an action (e.g. "A student")
  • mod: An optional modal for an action (e.g. "can" or "cannot")
  • action: The action performed by the subject (e.g "transfer")
  • object: The object on which the action is performed (e.g. "a proximity card")
  • secondary object: an optional object that modifies the action (e.g. "to another student")
  • purpose: an optional purpose for which the action is performed (e.g. "for building access grant")
  • condition: any optional condition on the action performed (e.g. "if the student is not authorized")
